## 改进建议

1. **紧急补充 #2 (Sizer 被架空)**：在 Phase 0 新增 Task 0.5，修改 `DualMaStrategy.generate_signals()` 移除硬编码的 `target_volume=100`，确保 `Signal` 不再携带交易数量，完全由 `PositionSizer` 接管。这是 P0 级遗漏。

2. **Phase 2 补充 #8 (多账户)、#10 (充血模型)、#12 (接口分裂)**：这三个 P2 缺陷在报告方向 A/B/C 中均有明确规划，应新增对应 Task。
   - Task 2.3: 多账户解耦 —— 引入 `AccountRepository`，消除 `"MOCK_ACCOUNT"` 硬编码
   - Task 2.4: BacktestReport 充血模型改造 —— 将 `PerformanceEvaluator` 中的计算逻辑内聚为 `BacktestReport` 的 property
   - Task 2.5: 重构 MarketGateway 接口 —— 将 `get_all_timestamps()` 从 `IMarketGateway` 剥离到 `IBacktestMarketGateway`

3. **Phase 3 补充 #13 (Portfolio)、#17 (错误处理) 和 #18 (SDK剥离)**：
   - Task 3.5: Portfolio 算法丰富 —— 凯利公式、等权重再平衡
   - Task 3.6: 领域异常体系 —— 创建 `InsufficientFundsError`、`PositionNotAvailableError` 等专用异常，替换宽泛 `except Exception`
   - Task 3.7: xtquant SDK 剥离 —— 将 `.pyd` 文件移出 `src/`，通过 `pip install -e .` 或外部路径引用

4. **建议调整优先级标注一致性**：将 Phase 1 的 Task 1.2 (涨跌停) 和 Task 1.3 (停牌/ST) 在 Plan 文档中明确标注为 "P0 业务"，与报告优先级保持一致。

5. **Plan 中 Task 3.4 (多策略支持) 缺乏具体实施步骤**：该 Task 仅在 Phase 3 概述中一句话提及 "改造 BacktestAppService 接受 list[BaseStrategy]"，缺乏类似 Phase 0 各 Task 的 Step 级详细设计，建议补充。
