🚀 快速开始

环境要求

  • Python 3.13+ (WSL conda 环境: goldenhandquant)
  • QMT 客户端 (MiniQMT) 已启动，路径: C:\QMT\userdata_mini
  • Windows Python 用于 QMT/xtquant 交互: /mnt/c/Users/11492/.conda/envs/goldenhandquant/python.exe

安装

conda activate goldenhandquant
pip install -e ".[dev,api,ml]"

❓ 常见问题

Q: QMT 连接失败怎么办？

确保 MiniQMT 客户端已启动，运行连接测试:

python scripts/test_qmt_connection.py

xtdata 会自动连接本地服务 (127.0.0.1:58610)，无需手动指定地址。

Q: WSL 中如何使用 xtquant？

xtquant 是 Windows 特定的包，必须使用 Windows Python:

/mnt/c/Users/11492/.conda/envs/goldenhandquant/python.exe -m src.interfaces.cli.fetch_quote -s 600519.SH

Q: quant ml-train 报 "unrecognized arguments" 错误？

已修复。现在可以直接使用:

python -m src.interfaces.cli.quant ml-train --symbols 600519.SH --start-date 2024-01-01 --end-date 2024-12-31

Q: 北向资金/龙虎榜数据不可用？

xtdata 未提供这些专用接口，命令会优雅降级返回提示。建议使用第三方数据源。

Q: 行业板块查询失败？

xtdata 的 get_sector_list() 只返回市场分类 (沪深A股等)，不支持行业板块 (半导体/新能源等)。

Q: ML 训练数据集如何构建？

自动构建: 直接运行 ml-train，如果数据集不存在会自动从 QMT 获取数据并构建 parquet 文件。

Q: 所有命令的输出格式？

数据采集命令输出统一 JSON 信封:

{
  "success": true,      // 或 false
  "data": { ... },      // 成功时的数据
  "error": null,        // 失败时的错误信息
  "timestamp": "..."    // ISO 8601 时间戳
}
