## 1. 背景与目标

### 1.1 背景

GoldenHandQuant 是一个 A 股量化投研团队，使用 Hermes Agent 的多 profile + kanban 系统协调多个 AI agent 完成投研任务。

v1.0 团队存在以下不足：
- 角色定义过于通用，缺乏专业投资哲学指导
- 没有多空辩论机制，容易产生单边偏见
- 没有专门的政策分析角色，不适应 A 股政策市特征
- 风险评估过于简单，缺乏多角度审视

### 1.2 目标

参考业界顶级开源项目（TradingAgents、ai-hedge-fund），将团队升级为"投资大师风格"：
- 每个核心分析角色体现一位投资大师的投资哲学
- 增加多空辩论机制，强制多空对抗
- 增加政策分析层，适应 A 股本土国情
- 增加三方风险评估，全面审视风险

### 1.3 参考项目

| 项目 | Stars | 核心特点 |
|------|-------|---------|
| TradingAgents | 高 | 7 角色、Bull/Bear 辩论、三方风险辩论、双模型策略 |
| ai-hedge-fund | 10k+ | 22 个投资大师 agent、结构化信号、评分体系 |

## 3. 各角色详细设计

### 3.1 quant-scout — 市场信息采集员

**定位**: 投研团队的"眼睛和耳朵"

**A 股特色数据维度**:
- 北向资金流向（净买入/卖出）
- 龙虎榜数据（机构/游资动向）
- 融资融券余额变化
- 大宗交易数据
- 限售解禁时间表
- 板块异动和轮动

**输出格式**: 结构化 Markdown 表格或 JSON

### 3.2 quant-analyst — 巴菲特价值分析师

**核心哲学**: "以合理价格买入优秀公司，以优秀价格买入普通公司。"

**分析框架**（参考 ai-hedge-fund/warren_buffett.py）:

#### 1. 护城河分析 (Moat Analysis) — 0-10分
| 维度 | 评分标准 | 权重 |
|------|---------|------|
| 品牌护城河 | 品牌溢价能力、消费者认知 | 0-2分 |
| 转换成本 | 客户切换成本、数据锁定 | 0-2分 |
| 网络效应 | 用户增长的价值、平台效应 | 0-2分 |
| 成本优势 | 规模经济、范围经济 | 0-2分 |
| 特许经营 | 牌照/专利/独家资源 | 0-2分 |

#### 2. 内在价值计算 (Intrinsic Value) — ¥人民币
- DCF 模型：自由现金流折现（10年预测 + 永续价值）
- 安全边际：(内在价值 - 当前价格) / 内在价值
- PE/PB 历史分位数

#### 3. 管理层质量 (Management Quality) — 0-10分
| 维度 | 评分标准 | 权重 |
|------|---------|------|
| 管理层持股 | 持股比例、增持/减持 | 0-3分 |
| 内部人交易 | 近 12 个月净买入/卖出 | 0-3分 |
| 薪酬挂钩 | 薪酬与 ROE/利润增长挂钩 | 0-2分 |
| 资本配置 | ROIC 趋势、分红/回购政策 | 0-2分 |

#### 4. 定价权 (Pricing Power) — 0-5分
- 毛利率趋势（3年）
- 提价能力（通胀转嫁测试）

#### 5. 基本面总分
- 总分 = 护城河 + 管理层 + 定价权（满分 25分）
- ≥18分 → bullish
- ≤8分 → bearish
- 其他 → neutral

**输出格式**:
```markdown
# 巴菲特价值分析报告：{标的}

## 护城河分析：{X}/10
{详细分析}

## 内在价值：¥{X}
- DCF 估值：¥{X}
- 安全边际：{X}%
- PE 分位数：{X}%

## 管理层质量：{X}/10
{详细分析}

## 定价权：{X}/5
{详细分析}

## 总分：{X}/25 → {bullish/bearish/neutral}
```

### 3.3 bull-researcher — 费雪成长分析师

**核心哲学**: "投资于拥有持续成长潜力的优秀公司。"

**分析框架**（参考 ai-hedge-fund/phil_fisher.py）:

#### 1. 成长潜力 (Growth Potential) — 0-10分
| 维度 | 评分标准 | 权重 |
|------|---------|------|
| 营收增长 | 3年/5年 CAGR | 0-4分 |
| 净利润增长 | 3年/5年 CAGR | 0-3分 |
| 市场份额 | 变化趋势 | 0-3分 |

#### 2. R&D 投入分析 — 0-5分
- R&D 占营收比例
- R&D 资本化 vs 费用化
- 专利/技术壁垒

#### 3. 行业渗透率 — 0-5分
- 当前渗透率
- 渗透率提升空间
- 行业天花板

#### 4. 管理层进取心 — 0-5分
- 扩张计划
- 并购整合能力
- 新业务布局

#### 总分: 0-25分 → 构建看多论据

### 3.4 bear-researcher — 伯里逆向分析师

**核心哲学**: "市场总是错的，关键是发现它错在哪里。"

**分析框架**（参考 ai-hedge-fund/michael_burry.py）:

#### 1. 估值泡沫分析 — 0-10分
| 维度 | 评分标准 | 权重 |
|------|---------|------|
| PE 分位数 | >80% = 高估 | 0-4分 |
| PEG | >2 = 泡沫信号 | 0-3分 |
| 市销率 | vs 行业平均 | 0-3分 |

#### 2. 财务风险识别 — 0-5分
- 负债率趋势
- 商誉/总资产比例
- 应收账款周转天数
- 现金流 vs 净利润背离

#### 3. 行业衰退信号 — 0-5分
- 行业增速放缓
- 产能过剩
- 竞争格局恶化

#### 4. 做空逻辑构建 — 0-5分
- 催化剂识别（业绩暴雷、政策变化）
- 下行空间量化
- 风险收益比

#### 总分: 0-25分 → 构建看空论据

### 3.5 macro-policy-analyst — 索罗斯宏观政策分析师

**核心哲学**: "市场参与者的偏见会影响市场价格，市场价格又反过来影响基本面。"（反身性理论）

**分析框架**:

#### 1. 财政政策分析 — 0-10分
| 维度 | 评分标准 | 权重 |
|------|---------|------|
| 基建投资 | 方向和力度 | 0-3分 |
| 减税降费 | 政策力度 | 0-2分 |
| 地方专项债 | 规模变化 | 0-3分 |
| 转移支付 | 方向 | 0-2分 |

#### 2. 货币政策分析 — 0-5分
- LPR 利率变化
- MLF/逆回购操作
- 存款准备金率调整
- 社融/M2 增速

#### 3. 监管风向研判 — 0-5分
- IPO 节奏变化
- 再融资政策
- 退市新规执行力度
- 量化交易监管

#### 4. 政策拐点识别 — 定性判断
- 政策底 vs 市场底
- 政策超预期/不及预期
- 政策传导时滞

#### A 股特色关注点
- 国家队动向（汇金、社保基金）
- 北向资金政策
- 印花税调整窗口
- 重要会议窗口期（两会、政治局会议、中央经济工作会议）

**输出格式**:
```markdown
# 宏观政策分析报告

## 政策信号：利好/利空/中性
{总结}

## 财政政策：{X}/10
{详细分析}

## 货币政策：{X}/5
{详细分析}

## 监管风向：{X}/5
{详细分析}

## 政策拐点
{定性判断}

## 影响行业
{行业列表}

## 影响标的
{股票列表}

## 政策强度：强/中/弱
## 持续性：长期/中期/短期
```

| 工信部政策 | 产业扶持力度 | 0-3分 |

#### 2. 行业准入/补贴分析 — 0-5分
- 行业准入门槛变化
- 财政补贴力度
- 税收优惠政策
- 政府采购方向

#### 3. 政策受益标的筛选 — 0-5分
- 行业龙头识别
- 政策弹性评估

#### 4. 政策风险预警 — 定性判断

### 3.7 risk-aggressive — 木木姐成长风险分析师

**核心哲学**: "高增长可以覆盖高估值，技术变革带来超额收益。"

**分析框架**:

#### 1. 颠覆性创新评估 — 0-10分
- 技术突破程度
- 商业模式创新
- 用户体验提升

#### 2. TAM 分析 — 0-5分
- 总可寻址市场
- 当前渗透率
- 增长空间

#### 3. 技术采用曲线 — 0-5分
- 当前阶段（创新者/早期采用者/早期大众/晚期大众）
- 加速拐点判断

**核心论点**:
- 高增长可以覆盖高估值
- 技术变革带来超额收益
- 不行动的风险（错过行情）

### 3.8 risk-conservative — 塔勒布反脆弱分析师

**核心哲学**: "保本比赚钱更重要，避免毁灭性亏损。"

**分析框架**:

#### 1. 尾部风险分析 — 0-10分
- 黑天鹅事件概率
- 最大回撤评估
- VaR 风险价值

#### 2. 反脆弱性评估 — 0-5分
- 公司在危机中的表现
- 现金储备充足度
- 业务多元化程度

#### 3. 凸性分析 — 0-5分
- 收益/风险的非线性关系
- 上行空间 vs 下行空间

**核心论点**:
- 保本比赚钱更重要
- 避免毁灭性亏损
- 凸性组合（有限亏损，无限收益）

### 3.9 risk-neutral — 芒格多元思维分析师

**核心哲学**: "长期复利思维，避免极端立场。"

**分析框架**:

#### 1. ROIC 分析 — 0-10分
- 投入资本回报率趋势
- ROIC vs WACC
- 资本配置效率

#### 2. 可预测性评估 — 0-5分
- 盈利可预测性
- 现金流稳定性
- 业务模式清晰度

#### 3. 多元思维模型 — 0-5分
- 心理学偏误检查
- 博弈论分析
- 复利效应评估

**核心论点**:
- 平衡收益和风险
- 长期复利思维
- 避免极端立场

### 3.10 portfolio-manager — 达里奥全天候基金经理

**核心哲学**: "风险平价，全天候配置。"

**决策框架**:

#### 1. 信号加权
| 信号来源 | 权重 |
|---------|------|
| 基本面信号（巴菲特） | 30% |
| 成长信号（费雪 vs 伯里辩论） | 25% |
| 政策信号（索罗斯 + 林园） | 25% |
| 风险信号（三方辩论） | 20% |

#### 2. 风险平价配置
- 根据波动率调整仓位
- 避免单一标的过度集中
- 保持组合分散度

#### 3. 最终决策
- Buy / Overweight / Hold / Underweight / Sell
- 目标价位（¥人民币）
- 建议仓位（%）
- 止损价位（¥人民币）
- 持有周期

### 3.11 secretary — 团队协调员

**定位**: 指挥中枢，连接用户与各专业 agent

**投研工作流 v2.0**:

```
第一阶段：数据采集（并行）
├── quant-scout: 行情、新闻、资金流向、龙虎榜
└── quant-repo-scout: 开源项目研究（可选）

第二阶段：投资大师分析
└── quant-analyst (巴菲特): 价值分析、内在价值计算

第三阶段：政策分析（并行）
├── macro-policy-analyst (索罗斯): 宏观政策研判
└── industry-policy-analyst (林园): 产业政策解读

第四阶段：多空辩论
├── bull-researcher (费雪): 基于数据+政策构建看多论据
└── bear-researcher (伯里): 基于数据+政策构建看空论据

第五阶段：风险评估（三方辩论）
├── risk-aggressive (木木姐): 高收益视角
├── risk-conservative (塔勒布): 风险控制视角
└── risk-neutral (芒格): 平衡视角

第六阶段：最终决策
└── portfolio-manager (达里奥): 综合所有意见，做出最终决策

第七阶段：汇报
└── secretary: 整理所有报告，汇报给用户
```

## 4. 辩论机制设计

### 4.1 多空辩论规则（参考 TradingAgents）

- **辩论轮数**: 默认 1 轮（可通过 kanban 配置 max_debate_rounds）
- **辩论流程**:
  1. bull-researcher 先发言，构建看多论据
  2. bear-researcher 回应，构建看空论据并反驳看多
  3. bull-researcher 再次回应，反驳看空
  4. bear-researcher 最终回应
  5. 双方论据汇总，提交给风险评估层

### 4.2 风险辩论规则

- **辩论轮数**: 默认 1 轮
- **辩论流程**:
  1. risk-aggressive 先发言
  2. risk-conservative 回应
  3. risk-neutral 平衡观点
  4. 三方论据汇总，提交给 portfolio-manager

## 7. 工程化设计

### 7.1 黑板架构（Blackboard Architecture）

系统采用事件驱动的黑板架构，Kanban 充当"内存数据库"：

```
┌─────────────────────────────────────────────────────────────────┐
│                    Hermes Kanban (状态总线)                       │
│                                                                  │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │ 行情数据  │  │ 财务数据  │  │ 政策分析  │  │ 辩论结果  │        │
│  │ JSON/MD  │  │ JSON/MD  │  │ JSON/MD  │  │ JSON/MD  │        │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘        │
│       ↑              ↑              ↑              ↑             │
│  quant-scout   quant-analyst   policy-agents   debaters         │
│       ↓              ↓              ↓              ↓             │
│  ┌──────────────────────────────────────────────────────┐       │
│  │           portfolio-manager (读取所有状态)              │       │
│  └──────────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────────┘
```

**关键设计原则**：
- 第一阶段数据采集结果以标准 JSON/Markdown 格式挂载在 Kanban task 的 workspace 中
- 后续 Agent 并发读取 State 数据，不再重复调用 API
- 避免每次调用都重复输入基础数据，节省 Token 且保证上下文一致性

### 7.2 并发执行设计

12 个 Agent + 多轮辩论需要高度异步并发：

```
时间线 →
────────────────────────────────────────────────────────────────
Phase 1: [quant-scout] ──────────────┐
         [quant-repo-scout] ─────────┤ (并行)
                                     ↓
Phase 2:                    [quant-analyst (巴菲特)]
                                     ↓
Phase 3: [macro-policy (索罗斯)] ────┤ (并行)
         [industry-policy (林园)] ───┘
                                     ↓
Phase 4: [bull (费雪)] ←→ [bear (伯里)]  (辩论，串行)
                                     ↓
Phase 5: [risk-aggressive (木木姐)] ──┐
         [risk-conservative (塔勒布)] ┤ (辩论，串行)
         [risk-neutral (芒格)] ───────┘
                                     ↓
Phase 6:                [portfolio-manager (达里奥)]
                                     ↓
Phase 7:                    [secretary 汇报]
────────────────────────────────────────────────────────────────
```

**Kanban 调度配置**：
- Phase 1、3 的任务设置 `parallel: true`，调度器自动并发
- Phase 4、5 的辩论任务设置 `depends_on` 依赖关系
- Phase 2 依赖 Phase 1 完成，Phase 4 依赖 Phase 2+3 完成

### 7.3 模型路由策略

并非所有角色都需要顶级模型，按任务复杂度路由：

| 任务类型 | 角色 | 推荐模型 | 理由 |
|---------|------|---------|------|
| 深度推理 | portfolio-manager, quant-analyst | Claude 3.5 Sonnet / GPT-4o | 最终决策需要最强推理 |
| 辩论推理 | bull/bear, risk-aggressive/conservative/neutral | Claude 3.5 Sonnet / GPT-4o | 辩论需要逻辑严密 |
| 信息提取 | quant-scout, secretary | Llama-3-8B / DeepSeek-Coder | 文本总结、格式化 |
| 政策分析 | macro/industry-policy | Claude 3.5 Sonnet | 政策解读需要推理 |

**config.yaml 配置示例**：
```yaml
# quant-scout 的 config.yaml
model:
  default: deepseek-coder  # 信息提取用小模型
  provider: local

# quant-analyst 的 config.yaml
model:
  default: claude-3.5-sonnet  # 深度推理用大模型
  provider: anthropic
```

### 7.4 评分锚点设计（Few-Shot Prompting）

LLM 打分存在"趋中性"偏误（喜欢打 7-8 分）。解决方案：在 SOUL.md 中提供锚点案例。

**quant-analyst 护城河分析锚点**：
```markdown
## 评分锚点（Few-Shot Examples）

### 10 分案例：贵州茅台
- 品牌护城河：国酒品牌，无可替代
- 转换成本：极高（社交场景绑定）
- 定价权：连续提价不影响销量
→ 护城河总分：10/10

### 2 分案例：某普通制造业公司
- 品牌护城河：无品牌溢价
- 转换成本：低（同质化竞争）
- 定价权：无（价格战）
→ 护城河总分：2/10

### 评分规则
- ROE > 20% 连续 3 年 + 垄断地位 → 10 分
- ROE 15-20% + 竞争优势明显 → 7-8 分
- ROE 10-15% + 一般竞争 → 4-6 分
- ROE < 5% + 价格战 → 1-3 分
```

**bear-researcher 估值泡沫锚点**：
```markdown
## 评分锚点

### 10 分案例：2021年某新能源股
- PE 历史分位数：95%
- PEG：3.5
- 市销率：行业平均的 5 倍
→ 泡沫总分：10/10

### 2 分案例：某银行股
- PE 历史分位数：15%
- PEG：0.8
- 市销率：行业平均的 0.3 倍
→ 泡沫总分：2/10
```

## 8. 风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| 角色过多导致协调复杂 | secretary 负责协调，kanban 管理任务依赖 |
| 辩论消耗过多 token | 默认 1 轮辩论，可配置；模型路由降低小任务成本 |
| 政策分析依赖实时数据 | 依赖 web 搜索和浏览器抓取；可对接 Tushare 结构化数据 |
| 评分趋中性/随机波动 | Few-Shot 锚点案例 + 硬性逻辑树 |
| 上下文不一致导致辩论脱节 | 黑板架构，State 统一存储，Agent 并发读取 |
| 策略回退困难 | Git 分支管理，--no-ff 合并保留演进节点 |
