## 三种失败模式

**1. IC 太弱/太抖（F01/F02/F03）**：方向全对、单调性近乎完美、IS/OOS 多空都是正的，
但日度 IC 的信噪比（IR）远低于 0.3 门槛——信号存在但不稳定，按日观察噪声占主导。
值得注意 F01 小市值的 OOS 多空 +41.9%（2024-07~2025-12 微盘风格大年），IR 低主因是
IS 段内含 2024 上半年微盘崩盘，时序波动巨大。

**2. 独立性不足（F03）**：低换手在剥离 log 市值 + 反转后残差 IC 仅 0.005——
它基本是小市值的影子因子，不提供独立信息。

**3. 截面排序对、做不成多空组合（F04/F05）**：这是最有信息量的失败。F04 低波动
IS 全指标通过（IC 0.053、IR 0.312、中性化后仍 0.051 有独立增量、单调性 1.0），
OOS 的 IC 也几乎没衰减（0.052），但 OOS 扣成本多空收益 −0.8%。排序信息真实存在，
却在 2024H2-2025 的行情里转化不成可落袋的多空价差（高波动腿在牛市段反弹强劲，
空头腿亏掉了多头腿的超额）。F05 同构，且 IS 多空也只有 +0.4%，基本是纯空头腿因子。

## 哪些东西是真的（跨因子证据）

- **五个因子 OOS IC 符号全部与 IS 一致**，无一翻转——不是数据挖掘出来的幻觉
- 单调性 4 个满分——分层结构整齐，截面排序信息真实
- 但"截面排序能力"≠"扣成本多空收益"，后者才是 edge 的定义，目前没有一个过线

## 已知缺陷（影响本轮判决解读，已排修复）

1. **`return_20d` 实现 bug**：`CrossSectionBuilder` 用 `closes[0]`（120 根窗口的第一根）
   而非 `closes[-21]`，实际是 ~119 日收益。直接后果：
   - F02 本轮实测的是"**半年期反转**"而非 20 日反转，真正的短期反转尚未被测过
   - 中性化控制变量 `return_20d` 同样错窗口 → F03 的"影子"判定、F04/F05 的
     "有独立增量"判定都需要修复后重验
2. 特征构建为纯 Python 逐日逐股重算，全市场一轮 ~2 小时——已决定建市场数据库
   （DuckDB）+ 向量化特征引擎，修复 bug 后以分钟级成本重跑干净判决
   （见 `docs/feat/0611-market-data-store/`）。
