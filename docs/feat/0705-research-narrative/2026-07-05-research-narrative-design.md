# 研究流水线叙事层 — 驾驶舱信息架构改造设计

> 状态: **待评审**（用户确认方向后实现）
> 起因: 用户反馈（2026-07-05, 原话）: "作为一个新手，还是无法使用和理解这些界面……因子判决还有回测中的数据从哪里来的，都是一些日期，根本看不懂是什么业务"

## 1. 问题诊断

前两轮修复解决了视觉层（对齐/降噪/术语解释），但新手仍看不懂——因为病根在**信息架构**：界面展示的是"系统的内部状态"，而非"用户的研究工作流"。

三个病灶：

| # | 病灶 | 症状（真实例子） | 后果 |
|---|------|----------------|------|
| P1 | **机器 ID 当身份** | 回测轮次叫 `20260614-103252`，判决轮叫 `MFCOMBO-20210101-20260613` | 用户只知道"何时跑的"，不知道"在验证什么想法"——"都是日期，看不懂业务" |
| P2 | **数据链路隐形** | 六个页签平铺，无先后、无流向；"判决的数据从哪来"无处可查 | 新手不知从哪看起，不理解页与页的关系 |
| P3 | **有解释无结论** | 悬停可查"IC 是什么"，但 F01 PASS 之后该干嘛、125% 是好是坏，无人话判词 | 看得懂每个数字，看不懂整体在说什么 |

**判定：业务本身有门槛（量化研究天然如此），但界面把门槛放大了。** 修复方向不是简化业务，而是给界面补上"叙事层"。

## 2. 业务事实（叙事层要讲的那条线）

本系统的全部业务逻辑就是一条流水线（每个页签正好是一环）：

```
① 数据资产            ② 因子判决              ③ 策略回测            ④ 实盘
   DuckDB 本地库    →    截面特征喂给因子检验  →   过闸因子组成策略   →   dry_run 纸面前向
   日线 646万行          IC/超额/样本外闸门        放回历史模拟收益        (影子盘攒样本中)
   截面特征 628万行      31 检验 / 1 过闸          29 轮
   ↑                                                                    ↓
   总览页              判决页                    回测页                实盘页
```

- 判决用的数据 = 总览页里的 `stock_features` 628 万行（T-1 口径预计算因子）
- 回测用的数据 = 总览页里的 `bars` 日线 + 截面策略再读 `stock_features`
- 回测的策略 = 判决过闸因子的工程化（如 F01 小市值 → MicroValueStrategy）

叙事层的任务：**把这条线画在界面上，并让每条记录用业务语言自我介绍。**

## 3. 方案（叙事层四件套）

### A. 研究流水线地图（总览页 hero，签名元素）

总览页标题下方新增一条可点击的流水线横条，四节点实时状态：

```
┌──────────────────────────────────────────────────────────────────────┐
│  ① 数据资产          ② 因子判决          ③ 策略回测        ④ 实盘     │
│  646万行 日线    →    1/31 过闸      →    29 轮回测    →    dry_run   │
│  行情与因子原料       谁有预测力          能否转化为收益     纸面前向    │
└──────────────────────────────────────────────────────────────────────┘
     点击跳总览KPI       点击跳判决页         点击跳回测页      点击跳实盘页
```

- 节点结构：序号+名称（display 字体）/ 状态数字（mono 大字）/ 一句职责（12px 灰）
- 节点间箭头附流向微标（"特征喂检验"、"过闸组策略"、"策略上纸面"）
- 数据源：**并行复用现有 4 个端点**（/research/overview、/research/verdicts 最新轮、/research/backtests、/live/overview），不新增后端聚合端点（复用别重算）；某端点失败该节点显示 "—" 不阻塞其他
- 双主题；当前页对应节点高亮 accent 左边框（与导航激活语言一致）

### B. Run 业务化命名（P1 根治）

展示层纯函数生成人话标题，机器 ID 降级为副行。**不改后端、不入库。**

回测轮次列表（左轨行改三行结构）：

```
现在:  20260614-103252          2026-06-14 10:32     →  改为:  小市值截面 · 全市场 · 2021→2026
       [MicroValueStrategy]                                     F01 可投性验证 · 06-14 10:32
                                                                20260614-103252 (11px 弱化)
```

命名规则：
- **策略中文名**：取 `/api/meta/strategies` 的 description 首短语（已有接口，避免前端硬编码漂移）；fallback 英文名
- **对象**：cross_section → `全市场`；否则 `首标的 +N`
- **区间**：`start_date→end_date` 取年份，如 `2021→2026`
- **来源映射**（前端小字典）：`f01_investability→F01 可投性验证`、`compare_strategies→多策略对比`、`web→网页提交`、`shadow_paper→影子盘`、`cli→CLI`

判决轮下拉同理：`31 因子 · 长多口径 · 切分 2024-06-30`，副行 `2026-06-14 10:02 · MFCOMBO-…`

### C. 数据血缘一句话（P2 补刀）

各页 meta 条加一项带 GlossaryTip 的血缘标注：

- 判决页：`数据 ← 本地库截面特征 · 5207 只`（tip: "检验用的因子值即总览页 stock_features 表, T-1 口径预计算"）
- 回测页：`行情 ← 本地日线库`；截面策略追加 `因子 ← 截面特征`
- 行情页已有（guide 已写"特征即因子检验用的原料"）

### D. 结论翻译（P3，克制版）

不加新行、不加噪音，在既有元素上补"所以呢"：

- 判决 PASS 徽章 tip 补一句：`→ 可进入策略回测`；FAIL 徽章 tip：`→ 淘汰, 红色项为未过原因`
- 回测超额既有红绿数字，措辞升级：`超额 +62.08%` → `跑赢基准 62.08%`（负值"跑输"）
- 流水线地图节点职责句（A 中已含）承担大部分"这环干嘛"的翻译

## 4. 决策点（请拍板）

| # | 问题 | 推荐 | 备选 |
|---|------|------|------|
| D1 | 流水线地图位置 | **总览页顶部**（总览=系统首页，承担地图职责） | 全局顶栏下常驻（太占空间，不推荐） |
| D2 | 策略中文名来源 | **meta description**（单一来源） | 前端映射表（会漂移） |
| D3 | 结论翻译深度 | **克制版**（徽章 tip + 超额措辞） | 每行加判词列（信息过载风险） |
| D4 | 后端聚合端点 | **不加**，前端并行拉现有 4 端点 | 加 /api/research/pipeline（多余抽象） |

## 5. 非目标

- 不做 onboarding 向导浮层/新手教程（guide 句 + 流水线地图 + tips 已构成渐进披露）
- 不改后端数据模型、不给 run 入库业务名（命名是展示层纯函数，规则可随时迭代）
- 不动任务页/行情页结构（本轮不涉及）

## 6. 验收标准（新手视角走查）

1. 打开总览 → 流水线地图 10 秒内看懂"系统四步、数据怎么流"
2. 点地图"因子判决"节点 → 落判决页，列表标题是业务人话，不再是纯日期
3. 判决页 meta 一眼看到"数据 ← 截面特征"，知道原料在总览
4. 回测左轨每行能读出"什么策略·什么对象·什么区间·谁提交的"
5. 常规验收链：vitest 全绿、6 页冒烟零 console 错误、双主题读图、构建入库

## 7. 实施拆分（确认后执行）

1. `pipeline` 组件 + 总览接入（A）——约半天
2. run 命名纯函数 + 回测左轨/判决下拉改造（B）+ 单测——约半天
3. 血缘标注 + 结论措辞（C/D）——1-2 小时
4. 读图验收 + 构建 + 提交
