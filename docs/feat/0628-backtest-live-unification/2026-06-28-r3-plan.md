## 二、方案 Y 设计决策（对 design DD-4 的优化）

design DD-4 原设想「改 `AutoTradeAppService` 消费 `OrderTarget`」，核实后发现**更优的方案 Y**：

### DD-Y1 · `AutoTradeAppService` 完全不改
它继续消费 `SignalDisplay`（含 confidence_score/suggested_price/suggested_volume/direction=SignalDirection）。**避免** agent 核实出的 OrderTarget 适配缺口（OrderTarget 缺 confidence_score、direction 类型不同、字段名 price/volume 不同）。

### DD-Y2 · `LiveSignalService.scan` 内部改用决策核心
`scan(strategy_name, symbols)` 改为：
```
config = get_strategy(strategy_name)          # 拿 strategy_type
runner = 构造 StrategyRunner(按 strategy_type: CrossSectionalStrategyRunner / SingleStrategyRunner)
         注入 self.market_gateway / self.trade_gateway / self.sizer / self.fundamental_registry
targets, prices = runner.evaluate(DayContext(current_time=now, symbols=symbols, base_timeframe=DAY_1))
return [_target_to_display(t, prices) for t in targets]
```
删除 `_scan_cross_sectional` / `_scan_bar` / `_build_fallback_snapshots`（决策重写 + market_cap=0 降级的根源）。

### DD-Y3 · `OrderTarget → SignalDisplay` 转换（补 UI 字段）
新 helper `_target_to_display(t: OrderTarget, prices: dict) -> SignalDisplay`:
```python
current = prices.get(t.symbol, t.price)
return SignalDisplay(
    symbol=t.symbol,
    direction=SignalDirection(t.direction.value),   # OrderDirection -> SignalDirection
    current_price=current,
    suggested_price=t.price,
    suggested_volume=t.volume,
    required_capital=round(t.price * t.volume, 2),
    reason=f"决策核心({t.strategy_name})",
    strategy_name=t.strategy_name,
    confidence_score=1.0,                            # 截面目标组合无 per-signal 置信度, 默认通过
)
```
保留现有 `_signals_to_displays`（半自动 live 的 place_confirmed_orders 仍用 SignalDisplay）。

### DD-Y4 · 两条实盘路径统一
全自动(`AutoTradeAppService.run_cycle` → `signal_service.scan`)与半自动(`live_trade.py` → `LiveSignalService.scan`)**都经此 scan** → 同时统一到决策核心(含趋势闸)。一处改动覆盖两条。

### DD-Y5 · wiring（R3c）
`_build_service` 改 `LiveSignalService` 构造，注入：
- `sizer`: 按 `at.strategy` 选 —— micro_value/截面 → `EqualWeightSizer(n_symbols=top_n)`；时序 → `FixedRatioSizer`(保持)。
- `fundamental_registry`: dry-run 经 `build_backtest_cross_section("DuckDBHistoryDataFetcher", ...)` 装(DuckDB 同源)；真实 QMT live 留下一 Spec。
- `risk_settings`: **不传**(`CrossSectionalStrategyRunner` 默认 index_symbol="000852.SH"/max_loss=0.03 即 F01 所需)。
- `market_gateway`: dry-run 影子盘(R4)可复用 `MockMarketGateway`(预装 DuckDB bars + set_current_time)；线上 dry-run 暂 QmtMarketGateway。
